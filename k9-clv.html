Project Overview

This is a real-world end-to-end analytics engagement on 3 years of operational data from a pet daycare and boarding facility in Bethlehem, PA. Starting from a raw 2.4GB MySQL backup exported from the Gingr pet business management platform, I built a cloud database on AWS RDS, cleaned and imported the data, and am now building a full CLV analysis to surface actionable insights for the business owners.

The business offers boarding (priced by room type with 20% multi-dog discount), flat-rate daycare, and grooming. The pricing structure creates natural customer segments with meaningfully different lifetime value profiles — the core hypothesis is that customers who use both boarding and daycare have significantly higher CLV than single-service customers. The data pipeline runs on AWS end-to-end: raw exports staged in S3, imported via Python on EC2, and loaded into MySQL on RDS — the same architecture pattern used in production data warehouses.

Infrastructure & Data Pipeline

The data pipeline runs entirely on AWS. The raw 2.4GB Gingr backup was staged in Amazon S3, processed and imported via a Python script running on an EC2 instance, and loaded into a MySQL database hosted on Amazon RDS.

Pipeline stages:

  • 01Raw Export: 2.4GB Gingr PMS backup exported and uploaded to S3
  • 02EC2 Processing: Python script on EC2 parses, cleans, and batches the raw dump
  • 03RDS Import: Cleaned records loaded into MySQL on AWS RDS via batch insert
  • 04Analysis Layer: Pandas/Python queries RDS directly for CLV and RFM modeling

This architecture keeps raw data in S3 for reproducibility, offloads compute-heavy import processing to EC2, and uses RDS as the persistent analytical database — mirroring a production data warehouse pattern at small scale.

Analysis Plan

  • 01Customer overview — total customers, new customer acquisition trend, active vs. churned breakdown
  • 02Core CLV model — AOV × frequency × tenure, CLV distribution, top 20% revenue concentration
  • 03RFM segmentation — recency, frequency, monetary scoring; Champions vs. At Risk vs. Lost
  • 04Service mix analysis — boarding only vs. daycare only vs. both; room tier revenue split
  • 05Package analysis — package buyer vs. drop-in CLV comparison; redemption rate
  • 06Cohort retention — monthly cohort table, retention curve, holiday boarding analysis
  • 07Seasonality — revenue by month, holiday concentration, boarding vs. daycare patterns
  • 08Breed analysis — CLV by breed, large vs. small breed revenue differences
